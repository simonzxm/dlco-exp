\section{实验目的}
本实验旨在设计并实现一个能够完成带符号补码数的加减法、取反、与、或、异或及比较大小等8种功能的4位 ALU，并在 FPGA 开发板上完成硬件验证与结果的数码管显示。

\section{环境配置}
本次实验的工程构建摒弃了传统的 Vivado GUI 繁重操作，采用了更为轻量高效的 \texttt{Makefile} 结合 \texttt{.tcl} 脚本的编译方式。开发者只需在终端通过执行标准的 \texttt{make} 指令，即可一键触发底层的 \texttt{.tcl} 脚本。该脚本自动涵盖了分析 Verilog 源文件、加载约束文件（\texttt{.xdc}）、逻辑综合生成网表、布局布线直至生成并烧录 Bitstream 文件的全流程。这种自动化脚本当大地加速了 FPGA 的上板调试与开发迭代效率。

\section{实验原理与设计}
实验的核心部分通过 \texttt{alu\_s} 模块实现，该模块利用 3 位控制信号 \texttt{ALUctr} 控制输出。进行加减法时，同步计算零标志位（\texttt{zero}）、溢出标志位（\texttt{of}）和进位标志位（\texttt{cf}）。其中，\texttt{cf} 标志在加法时表示最高位产生的进位（Carry Out）；在减法时，它则表示被减数小于减数时产生的借位（Borrow Out）。同时，在实现比较大小时，调用了 Verilog 的内置系统函数 \texttt{\$signed()} 对操作数进行强制类型转换，确保底层比较逻辑按照带符号补码的标准执行，避免了默认无符号比较带来的逻辑漏洞。

在顶层模块 \texttt{top} 中，使用开发板上的拨码开关作为操作数A、 B的输入与控制信号，ALU的计算结果经拆分后传至 \texttt{display} 和 \texttt{bcd7seg} 模块，驱动七段数码管动态显示。状态标志位则由 LED 输出。

\section{核心代码展示}
核心功能基于多路选择器 \texttt{case} 语法实现：
\begin{lstlisting}[language=Verilog]
always @(*) begin
    F = 4'd0; cf = 1'b0; of = 1'b0; zero = 1'b0;
    case (ALUctr)
        3'b000: begin
            {cf, F} = A + B;
            of = (A[3] == B[3]) && (F[3] != A[3]);
            zero = ~(|F);
        end
        3'b001: begin
            {cf, F} = A - B;
            of = (A[3] != B[3]) && (F[3] != A[3]);
            zero = ~(|F);
        end
        3'b010: F = ~A;
        3'b011: F = A & B;
        3'b100: F = A | B;
        3'b101: F = A ^ B;
        3'b110: F = $signed(A) < $signed(B);
        3'b111: F = (A == B);
    endcase
end
\end{lstlisting}

\section{实验总结}
本实验成功实现了一个支持复杂逻辑与算术运算的4位补码 ALU。经数码管及 LED 验证，计算结果及进位、溢出表示准确无误。本次实验加深了对补码运算特性及 FPGA 硬件模块化设计的理解。
