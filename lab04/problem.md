## 4.5 实验内容

## 4.5.1 上板验收

请在一个工程中完成如下的寄存器堆和RAM。寄存器堆和RAM的大小均为16x8，即都有16个存储单元，每个单元都是8位的，均可以进行读写。

- **寄存器堆：** 读取时不需要时钟控制，即读地址有效后，直接输出数据。写入时通过时钟上升沿进行控制。

此时可用以下方式输出:

```verilog
assign out = ram [addr];
```

采用下面的方式进行初始化。

```verilog
initial
begin
    $readmemh("D:/digital_logic/meml.txt", ram, 0, 15);
end
```

初始化数值为

```
@0 00
@1 01
@2 02
@3 03
@4 04
@5 05
@6 06
@7 07
@8 08
@9 09
@a 0a
@b 0b
@c 0c
@d 0d
@e 0e
@f 0f
```

- **RAM：** 利用IP核设计一个单口存储器，利用.coe文件进行初始化，十六个单元的初始化值分别为: 0xf0, 0xf1, 0xf2, 0xf3, 0xf4, 0xf5, 0xf6, 0xf7, 0xf8, 0xf9, 0xfa, 0xfb, 0xfc, 0xfd, 0xfe, 0xff。

此两个物理上完全不同的存储器共用时钟、读写地址。适当选择时钟信号和写使能信号，以能够分别对此两个存储器进行读写。请将两个存储器读出的结果分别用2个七段数码管显示。请合理使用FPGA开发板的输入/输出资源，完成此寄存器堆和RAM的设计。由于开发板上输入数量不够，写入时可以只写入4位数据。

- 请使用开发板上的按钮来做为存储器的时钟信号。观察两个不同的实现方式下各需要几个时钟周期来完成读取或写入操作?
- 在实现之后 Open Implementation Design，观察 Schematic 中你生成的两个存储器，点击存储器模块直至最内层，观察综合后这两个存储器分别使用了什么方式来实现，为什么?
- 注意观察综合后输出的资源消耗，在 Implement 之后选择 Report Utilization，图4-12中显示我们用IP核生成的RAM使用了多少Block RAM？观察寄存器堆使用了哪些资源。
