\section{寄存器堆设计}

本实验中的寄存器堆为 16$\times$8 位结构，采用 \textbf{异步读、同步写} 的设计方式。读取时通过组合逻辑直接输出，写入则在时钟上升沿完成。

\begin{lstlisting}
module regfile (
    input        clk,
    input        we,
    input  [3:0] addr,
    input  [7:0] wdata,
    output [7:0] rdata
);
  reg [7:0] ram [0:15];

  initial begin
    $readmemh("mem1.txt", ram, 0, 15);
  end

  assign rdata = ram[addr];           // 异步读

  always @(posedge clk) begin
    if (we) ram[addr] <= wdata;       // 同步写
  end
endmodule
\end{lstlisting}

初始化文件 \texttt{mem1.txt} 使用 \texttt{@地址 数据} 格式，将地址 0--15 分别初始化为 \texttt{0x00}--\texttt{0x0F}。

异步读的关键在于 \texttt{assign} 语句：地址信号一旦改变，输出立即通过组合逻辑更新，\textbf{无需时钟沿}。综合后，Vivado 会将这种结构映射为分布式 RAM（Distributed RAM），使用 FPGA 的 LUT 资源实现。

\section{RAM 设计（IP 核）}

RAM 部分利用 Vivado 的 Block Memory Generator IP 核生成一个 16$\times$8 的单口存储器。由于本项目不使用 Vivado GUI，IP 核通过 Tcl 脚本自动创建：

\begin{lstlisting}[language=Tcl]
create_ip -name blk_mem_gen -vendor xilinx.com \
    -library ip -version 8.4 \
    -module_name blk_mem_gen_0 -dir $IP_DIR

set_property -dict [list \
    CONFIG.Memory_Type {Single_Port_RAM} \
    CONFIG.Write_Width_A {8} \
    CONFIG.Write_Depth_A {16} \
    CONFIG.Register_PortA_Output_of_Memory_Primitives {false} \
    CONFIG.Load_Init_File {true} \
    CONFIG.Coe_File [file normalize ./ram_init.coe] \
] [get_ips blk_mem_gen_0]
\end{lstlisting}

初始化使用 \texttt{.coe} 文件，将 16 个单元分别初始化为 \texttt{0xF0}--\texttt{0xFF}。

与寄存器堆不同，IP 核生成的 RAM 为 \textbf{同步读写}：读取数据需要一个时钟上升沿才能输出到 \texttt{douta} 端口。综合后，Vivado 将其映射为 Block RAM（BRAM），使用 FPGA 内嵌的 RAMB18E1 硬件原语实现。

\section{顶层模块与 I/O 设计}

顶层模块将寄存器堆和 RAM 共用同一组地址开关和时钟按钮，通过一个选择开关区分写入目标：

\begin{lstlisting}
  // SW[13] 为写使能，SW[12] 为选择开关
  wire reg_we = SW[13] & ~SW[12];
  wire ram_we = SW[13] &  SW[12];

  wire [7:0] reg_rdata;
  regfile regfile_inst (
      .clk  (BTNC),
      .we   (reg_we),
      .addr (SW[3:0]),  // SW[ 3:0] 为地址信号
      .wdata(SW[11:4]), // SW[11:4] 为写入数据
      .rdata(reg_rdata)
  );

  wire [7:0] ram_rdata;
  blk_mem_gen_0 ram_inst (
      .clka (BTNC),
      .wea  (ram_we),
      .addra(SW[3:0]),  // SW[ 3:0] 为地址信号
      .dina (SW[11:4]), // SW[11:4] 为写入数据
      .douta(ram_rdata)
  );
\end{lstlisting}

开发板按钮 \texttt{BTNC} 直接作为两个存储器的时钟信号，每按一次产生一个时钟周期。由于 \texttt{BTNC} 并非专用时钟引脚，XDC 中需设置：
\begin{lstlisting}[language=Tcl]
set_property CLOCK_DEDICATED_ROUTE FALSE [get_nets BTNC_IBUF]
\end{lstlisting}

数码管显示方面，使用一个统一的 8 位动态扫描驱动 \texttt{display}，右侧 4 位（AN[3:0]）显示寄存器堆读出值，左侧 4 位（AN[7:4]）显示 RAM 读出值。译码器 \texttt{hex7seg} 支持 0--F 的十六进制显示。

\section{实验观察与分析}

\subsection{读取时钟周期差异}

\begin{itemize}
    \item \textbf{寄存器堆}：改变地址开关后，右侧数码管\textbf{立即}更新显示。读取需要 \textbf{0 个时钟周期}（异步读，纯组合逻辑）。
    \item \textbf{RAM（IP 核）}：改变地址开关后，左侧数码管\textbf{不变}，必须按一次 BTNC 才能读出新数据。读取需要 \textbf{1 个时钟周期}（同步读）。
\end{itemize}

\subsection{综合后的实现方式}

通过 Vivado 的 Primitives 报告，可以直接观察到两个存储器使用了不同的硬件原语：

\begin{center}
\begin{tabular}{|l|l|l|}
\hline
\textbf{存储器} & \textbf{使用的原语} & \textbf{实现方式} \\
\hline
寄存器堆 (\texttt{regfile\_inst}) & RAMS32 $\times$ 4 & Distributed RAM（LUT） \\
\hline
RAM (\texttt{ram\_inst}) & RAMB18E1 $\times$ 1 & Block RAM \\
\hline
\end{tabular}
\end{center}

\textbf{寄存器堆}使用了 4 个 \texttt{RAMS32}（Single-port Distributed RAM 原语），属于 Distributed Memory 类别。这是因为 Verilog 代码中使用了 \texttt{assign} 实现异步读取，这种组合逻辑行为只能用 LUT 来实现，Vivado 自动将其综合为分布式 RAM。4 个 \texttt{RAMS32} 各提供 32$\times$1 的存储，组合起来实现 16$\times$8 的地址空间（每个 RAMS32 负责 2 位输出数据）。

\textbf{RAM（IP 核）}使用了 1 个 \texttt{RAMB18E1}，属于 Block Memory 类别。Block RAM 是 FPGA 芯片内嵌的专用存储硬件，天然支持同步读写操作。16$\times$8 仅需要 128 bit 的存储，远小于一个 RAMB18E1 提供的 18Kbit 容量，因此只使用了 1 个。

\subsection{资源消耗}

根据 Vivado 的 Report Utilization，各模块的资源占用如下：

\begin{center}
\begin{tabular}{|l|c|c|c|c|c|}
\hline
\textbf{模块实例} & \textbf{Total LUTs} & \textbf{LUTRAMs} & \textbf{FFs} & \textbf{RAMB18} \\
\hline
\texttt{top}（整体） & 19 & 4 & 17 & 1 \\
\hline
\texttt{regfile\_inst} & 5 & 4 & 0 & 0 \\
\hline
\texttt{ram\_inst} & 0 & 0 & 0 & 1 \\
\hline
\texttt{display\_inst} & 13 & 0 & 17 & 0 \\
\hline
\end{tabular}
\end{center}

从表中可以清晰看到：
\begin{itemize}
    \item IP 核生成的 RAM 使用了 \textbf{1 个 RAMB18}（占 Block RAM Tile 的 0.5 个，利用率 0.37\%），\textbf{不消耗任何 LUT 资源}。
    \item 寄存器堆使用了 \textbf{4 个 LUTRAM + 1 个逻辑 LUT}，\textbf{不消耗任何 Block RAM}，也不使用触发器（FF = 0），因为异步读不需要寄存器来锁存输出。
    \item 数码管显示模块消耗了最多的逻辑资源（13 个 LUT 和 17 个 FF），主要用于动态扫描计数器和多路选择逻辑。
\end{itemize}

这体现了两种存储实现的本质区别：分布式 RAM 灵活、支持异步读取，但消耗通用 LUT 资源；Block RAM 高效、容量大，但只能同步访问。
